ТЕМА: Формат передачи SCN в SNMP запросах (контроллер Синтез, протокол UG405)

───────────────────────────────────────────────────────────────

Здравствуйте!

Работаю с интеграцией контроллера светофора Синтез по протоколу UG405 через SNMP.

ПРОБЛЕМА:

В документации указано, что многие OID требуют передачи SCN (Site Control Number) 
в формате "+ SCN", например:
- SetPhase: 1.3.6.1.4.1.13267.3.2.4.2.1.5 + SCN
- SetAF (желтое мигание): 1.3.6.1.4.1.13267.3.2.4.2.1.20 + SCN

НО:
- Операции БЕЗ SCN работают (GET и SET)
- Операции С SCN в формате ASCII кодов в OID не работают

ВОПРОСЫ:

1. Как именно передается SCN в SNMP запросах?
   - В OID как ASCII коды? (BASE_OID.1.67.79.49.49.49.49.49)
   - В OID как число? (BASE_OID.1.11111)
   - В значении varbind?
   - Как отдельный varbind?
   - Другой формат?

2. Обязателен ли SCN для SET операций?
   - Может быть, контроллер работает и без SCN?

3. Есть ли примеры успешных SNMP запросов с SCN?
   - Примеры OID
   - Примеры кода (любой язык)

ТЕХНИЧЕСКИЕ ДЕТАЛИ:

- Контроллер: Синтез
- Протокол: UG405
- SCN: CO11111 (формат COX)
- MIB: UTMC-UTMCFULLUTCTYPE2-MIB

ЧТО РАБОТАЕТ БЕЗ SCN:
✅ GET Operation Mode: 1.3.6.1.4.1.13267.3.2.4.1
✅ GET Phase: 1.3.6.1.4.1.13267.3.2.5.1.1.3
✅ SET Phase: 1.3.6.1.4.1.13267.3.2.4.2.1.5 = 0x01
✅ SET AF: 1.3.6.1.4.1.13267.3.2.4.2.1.20 = 1

ЧТО НЕ РАБОТАЕТ С SCN (ASCII коды):
