ПРОМПТ ДЛЯ ПОИСКА ИНФОРМАЦИИ ПО ФОРМАТУ SCN

═══════════════════════════════════════════════════════════════

ЗАДАЧА:
Определить правильный формат передачи SCN (Site Control Number) 
в SNMP запросах для контроллера Синтез по протоколу UG405.

ТЕКУЩАЯ ПРОБЛЕМА:
- SET операции с SCN в формате ASCII кодов не работают
- SET операции БЕЗ SCN работают (но документация требует SCN)
- GET операции БЕЗ SCN работают

ЧТО НУЖНО НАЙТИ:

1. ИСХОДНЫЙ КОД (Node.js):
   - Файл spectr_utmc.js или аналогичный
   - Как формируется OID для SetPhase/SetAF
   - Как передается SCN в запросах
   - Примеры кода с использованием SCN

2. ПРИМЕРЫ РЕАЛЬНЫХ ЗАПРОСОВ:
   - Логи успешных SNMP запросов
   - Примеры OID с SCN из работающего кода
   - Wireshark дампы SNMP трафика

3. ДОКУМЕНТАЦИЯ:
   - Руководство по формату SCN
   - Примеры использования SCN
   - Техническая документация контроллера

4. АЛЬТЕРНАТИВНЫЕ ФОРМАТЫ (для тестирования):
   - SCN как ASCII коды: BASE_OID.1.67.79.49.49.49.49.49 (не работает)
   - SCN как число: BASE_OID.1.11111 (только цифры)
   - SCN в значении varbind (не в OID)
   - SCN как отдельный varbind в SET запросе

ВОПРОСЫ ДЛЯ ИССЛЕДОВАНИЯ:
1. Как именно передается SCN в OID?
2. Нужно ли преобразование SCN перед передачей?
3. Может быть, SCN передается не в OID, а в значении?
4. Есть ли примеры успешных запросов с SCN?

КОМАНДЫ ДЛЯ ПОИСКА:
# Поиск в коде
grep -r "utcControlFn\|siteId\|SCN" /path/to/code

# Поиск в логах
grep -i "snmp\|oid\|scn" /path/to/logs/*.log

# Поиск в MIB
grep -A 20 "utcControlSiteID\|INDEX" docs/UTMC-UTMCFULLUTCTYPE2-MIB*.txt
